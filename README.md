# Проект 27_01_2025

## Проект 27_01_2025 это приложение для работы с банковскими картами, счетами и данными клиентов

## Установка:

1. Клонируйте репозиторий:
'''
git clone https://github.com/Valentina1764/BVG/tree/develop
'''
2. Установите зависимости:
'''
pip install  -r requirements.txt
'''

## Использование:
В консоле выполните команду:
'''
python main.py
'''

## Функции, реализованные в проекте:
1. Модуль "masks":
   - Функция "get_mask_card_number" - возвращает замаскированный номер карты. Пример: 
    номер карты на входе "1596837868705199" на выходе "1596 83** **** 5199".
   - Функция "get_mask_account" - возвращает замаскированный номер счета. Пример: 
     номер счета на входе "73654108430135874305" на выходе "** 4305".
2. Модуль "widget":
   - Фукция "mask_account_card" - маскирование номеров карт и счетов.
   На входе:
    "Maestro 1596837868705199"
    "Счет 64686473678894779589".
   Результат:
    "Maestro 1596 83** **** 5199"
    "Счет **9589"
   соотвественно.
   - Функция "get_date" возвращает текущую дату в формате ДД.ММ.ГГГГ.
3. Модуль "processing":
   - Функция "filter_by_state" возвращает список словарей в зависимости от значения ключа "state".
   - Фукция "sort_by_date" возвращает отсортированный список по ключу "date".
4. Модуль  "generators":
   - Функция "filter_by_currency" возвращает список трансакций по типу валюты.
   - Функция "transaction_descriptions" возвращает значение ключа "description" из списка словарей трансакций.
   - Функция "card_number_generator" - генератор номеров кредитных карт от "0000 0000 0000 0001" до "9999 9999 9999 9999"
5. Модуль "my_decoracors":
   -  Декоратор простых функций сложения и деления.
6. Модуль "utils":
   - Содержит функцию "load_transactions", которая считывает данные из файла типа JSON и преобразует в объект Python
7. Модуль "external_api":
   - Функция "converter_currency" конвертирует валюту с помощью внешних API сервисов
8. Модуль "finacial_operation":
   - Функция "read_csv_transactions" принимает на вход файл c информацией о банковских операциях в формате: '.CSV'.
     Возвращает список словарей банковских операций.
   - Функция "read_excel_transactions" принимает на вход файл c информацией о банковских операциях в формате EXCEL(.xlsx).
     Возвращает список словарей банковских операций.
9. Модуль "operations":
   - Функция "search_operations_by_description" реализует поиск по списку трансакций с выбранным пользователем значением
   ключа "state".
   - Функция "search_operations_by_currency" реализует поиск по списку трансакций с выбранным пользователем значением
   ключа "currency_code".
   - Функция "search_operations_by_word" реализует поиск по списку трансакций с выбранным пользователем словом
   для ключа "description".
   - Функция "count_categories" подсчитывает количество банковских операций определенного типа.
10. Модуль "main":
   - Функция "main" ведет диалог с пользователем, импортирует списки словарей транзакций из файла в зависимости
   от выбора пользователя, передает данные во внешние функции для обработки и форматирует для вывода в консоль
   полученные результаты.

## Ввод данных:
Данные вводятся из файлов "card_account.txt","states.txt","transations.txt" и "operations.json"в дериктории "Data"

## Вывод результатов:
Результаты обработки выводятся в консоль

## Создан декоратор:
Декоратор автоматически регистрирует детали выполнения функций.

## Логирование моделей utils и masks
В модули 'utils' и 'masks' добавлены отдельные объекты логера.
Логи выводятся в соответствующие файлы 'utils.log' и 'masks.log' в директорию 'logs'.
